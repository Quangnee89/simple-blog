4. Database schema
Tạo đầy đủ SQL để chạy trong Supabase SQL Editor cho các thành phần sau:

4.1. Bảng `profiles`
- `id uuid primary key references auth.users on delete cascade`
- `display_name text`
- `avatar_url text`
- `created_at timestamptz default timezone('utc', now()) not null`
- `updated_at timestamptz default timezone('utc', now()) not null`

4.2. Enum `post_status`
- gồm hai giá trị: `draft`, `published`

4.3. Bảng `posts`
- `id uuid primary key default gen_random_uuid()`
- `author_id uuid not null references public.profiles on delete cascade`
- `title text not null`
- `slug text not null unique`
- `content text`
- `excerpt text`
- `status post_status default 'draft' not null`
- `created_at timestamptz default timezone('utc', now()) not null`
- `updated_at timestamptz default timezone('utc', now()) not null`
- `published_at timestamptz`

4.4. Bảng `comments`
- `id uuid primary key default gen_random_uuid()`
- `post_id uuid not null references public.posts on delete cascade`
- `author_id uuid not null references public.profiles on delete cascade`
- `content text not null`
- `created_at timestamptz default timezone('utc', now()) not null`

4.5. Indexes, comments, và relationship
- Tạo index phù hợp như trong đề cho các trường hay query
- Có SQL rõ ràng, sạch, chạy liền mạch [file:1]

5. Functions và triggers

- `update_updated_at()` để cập nhật trường `updated_at`
- trigger cho `profiles`

6. Row Level Security (RLS)
Phải bật RLS cho:
- `profiles`
- `posts`
- `comments`

Phải viết đầy đủ policies như sau:

6.1. `profiles`
- ai cũng có thể `select`
- authenticated user chỉ `update` profile của chính mình

6.2. `posts`
- ai cũng xem được bài `published`
- author xem được tất cả bài của mình kể cả `draft`
- authenticated user được `insert` post
- author chỉ được `update` post của mình
- author chỉ được `delete` post của mình

6.3. `comments`
- ai cũng xem được comments thuộc bài viết đã `published`
- authenticated user được `insert` comment
- user chỉ được `delete` comment của mình [file:1]

Ngoài ra:
- Nếu hợp lý, thêm phần SQL test RLS hoặc ghi chú cách test trong SQL Editor.
- Không thay đổi logic phân quyền gốc của đề nếu không có lý do rất rõ ràng.

7. Authentication
Phải triển khai đầy đủ flow xác thực:

7.1. Register
- route: `/register`
- form gồm:
  - display name
  - email
  - password
- dùng `supabase.auth.signUp`
- truyền `display_name` vào user metadata
- xử lý loading/error rõ ràng
- chuyển hướng sau khi đăng ký thành công [file:1]

7.2. Login
- route: `/login`
- login bằng email/password với `supabase.auth.signInWithPassword`
- login bằng GitHub với `supabase.auth.signInWithOAuth`
- có loading/error rõ ràng [file:1]

7.3. OAuth callback
- route: `/auth/callback`
- lấy `code` từ query params
- exchange code để tạo session
- redirect về `/dashboard` [file:1]

7.4. Logout
- tạo server action logout
- sign out khỏi Supabase
- redirect về `/login` [file:1]

7.5. Header
- có header dùng chung trong layout
- nếu chưa đăng nhập: hiện link Trang chủ, Đăng nhập, Đăng ký
- nếu đã đăng nhập: hiện Trang chủ, Dashboard, Đăng xuất [file:1]

8. CRUD bài viết
Phải triển khai đầy đủ CRUD cho `posts`:

8.1. Trang chủ `/`
- public
- hiển thị danh sách bài viết đã `published`
- join với `profiles` để hiện tên tác giả
- sắp xếp theo `published_at` giảm dần [file:1]

8.2. Dashboard `/dashboard`
- protected
- hiển thị toàn bộ bài viết của user hiện tại, gồm cả draft
- có nút tạo bài mới
- có danh sách bài viết với trạng thái draft/published [file:1]

8.3. Tạo bài mới `/dashboard/new`
- protected
- form gồm:
  - title
  - excerpt
  - content
  - status
- khi status là `published` thì set `published_at`
- author_id phải là user hiện tại [file:1]

8.4. Chỉnh sửa bài viết `/dashboard/edit/[id]`
- protected
- chỉ author mới được sửa
- query đúng theo `id` và `author_id`
- nếu không có quyền hoặc không tồn tại thì xử lý notFound hợp lý [file:1]

8.5. Xóa bài viết
- nút xóa ở dashboard
- xác nhận trước khi xóa
- chỉ author được xóa nhờ RLS [file:1]

8.6. Chi tiết bài viết `/posts/[slug]`
- public
- chỉ hiển thị bài `published`
- hiển thị title, author, publish date, content
- render nội dung theo cách đơn giản, rõ ràng; nếu cần có thể render từng paragraph theo newline như đề
- có generateMetadata nếu phù hợp [file:1]
